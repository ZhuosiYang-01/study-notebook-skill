# 调试与维护技巧

补充 SKILL.md 的「Gotchas」——记录次要的调试和维护技巧，按需查阅。

## "看不到某部分内容"时先怀疑 CSS 布局

最常见的不是 JS / 数据 / 渲染逻辑错误，而是 `overflow`、`max-height`、`position` 这类 CSS 布局问题截断了内容。**用"元素检查器"式思路先查布局，再查数据和逻辑**——顺序反了会浪费大量调试时间。

典型例子：侧边栏看不到第 5 章及以后的内容 → 90% 是 `.sidebar { overflow: hidden }` 截断了，应该改成 `overflow-y: auto`，**不是数据没渲染**。

## html 字段里注意转义反引号和 ${...}

chapters 里 block 的 html 字段是 JS 模板字符串（用反引号 `` ` `` 包裹），里面如果原样出现反引号或 `${...}` 会打断字符串、导致 JS 报错。

转义方式：

- 反引号 → 用 `` \` `` 转义
- `${...}` → 用 `\${...}` 转义
- 或者：把这些字符替换成视觉相近的中文标点（比如把 `${var}` 写成"$（var）"）

## 调试时不要新建版本化文件

调试渲染问题、想"做一个干净版"对比、或修复 CSS 时——都不要新建 `-v2.html` / `-fixed.html` / `-debug.html`。在原文件上 str_replace 改即可。

新建版本化文件的代价：
- 用户不知道哪个是当前版本
- 新文件 storage key 不同，之前的 myEdits/myNotes 全部失效
- 文件管理混乱

唯一例外：用户**明确请求**"做一个新版本保留旧版"——必须先确认。

## 笔记 HTML 加载失败时的常见原因

如果用户报告"笔记打不开 / 一片空白"，按下面顺序排查：

1. **JS 语法错误**——chapters 数据里有未转义的反引号、引号嵌套、缺逗号
2. **`<style>` 没闭合**——CSS 改动时漏了 `}` 会让后面 HTML 全被吞掉
3. **`</script>` 或 `</body>` 缺失**——HTML 不完整

最快定位方法：浏览器打开后按 F12 看 Console 报错信息，第一行错误通常就是真因。

## 编辑模式下用户改了内容但保存后看不到

可能原因：
- 用户改完没按 Ctrl+S 也没切回浏览模式（编辑模式下不会自动保存）
- 是在登录态过期后做的修改，storage 写入失败——刷新页面重新登录后再改一次

不要直接帮用户重写，而是先告诉他检查右上角是否有"已保存"提示，没有就手动按一次 Ctrl+S。
